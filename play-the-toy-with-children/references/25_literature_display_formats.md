# Literature Display Formats

Use this reference when turning literature-loop outputs into readable review
artifacts, proposal background figures, slide motivation panels, or paper
related-work structure.

## Core Displays

Landscape summary:

- Field clusters.
- Representative works.
- What is solved.
- What remains open.
- Why this project exists.

Literature cluster map:

- Cluster name.
- Method family.
- Correctness/trust mechanism.
- Evidence type.
- Relation to our method.
- Main limitation.

Reviewer-comparison table:

- Likely reviewer question.
- Literature behind the question.
- Our answer.
- Evidence.
- Risk and response.

Timeline:

- Use only when chronology clarifies the field.
- Mark review papers, method-defining papers, benchmark papers, and recent
  competing work separately.

Method taxonomy:

- Use when the field has adjacent method families.
- Columns should include what each family optimizes, what supplies
  correctness/trust, and where it fails.

Author/lab/method-lineage map:

- Use when one paper sits inside a continuing research program, lab platform,
  instrument family, dataset, benchmark, or method lineage.
- Separate "same authors or same lab" from "same method family by other groups".
- Mark earlier infrastructure, seed paper, direct follow-up, adjacent sibling
  work, later extension, and monitor-only work.
- Show what each lineage item changes: novelty, baseline, feasibility,
  reviewer risk, or future-work direction.

Problem -> method -> evidence funnel:

- Start with the practical/scientific problem.
- Show prior method families.
- Show the unresolved gap.
- Show our mechanism and evidence.
- End with the scope boundary.

## Visual Candidate Rules

- Every visual display must trace back to literature matrix rows.
- Do not make a taxonomy that hides a direct baseline.
- Do not turn speculative gaps into established facts.
- Use "open risk" labels for unresolved or weakly supported comparisons.
- For public slides/posters, redraw concepts rather than screenshotting full
  copyrighted figures unless permissions and context are clear.

## Output Contract

Return display candidates with:

- Display name.
- Purpose.
- Source rows used.
- Artifact target.
- Claim supported.
- Risk or missing source.
